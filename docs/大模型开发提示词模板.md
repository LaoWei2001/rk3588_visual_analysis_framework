# 大模型功能开发 · 提示词模板

> 用途：基于本项目（`rk3588_visual_analysis_framework`）用大模型快速开发新功能时，**不用每次从零组织提示词**。
> 从下面挑一个模板 → 把 `【】` 里的内容换成你的需求 → 整段复制给大模型即可。模板里**固定的部分**（要读哪些文档、必须遵守哪些约定、要产出什么、怎么验证）已经替你写好，你只管填需求。

---

## 30 秒上手

1. **选模板**：看下面《我要做什么 → 用哪个模板》路由表。绝大多数"检测到 X 就做 Y"的需求 → **模板 A**。
2. **填空**：复制该模板整段，把每个 `【...】` 换成你的内容（没有的项写"无"或删掉那行）。
3. **发给大模型**：
   - 用 **Claude Code / 有本仓库读写权限的 Agent**（推荐）：直接粘贴，它会自己去读 `docs/skills/` 里的文档再动手。
   - 用**普通网页版大模型**（无法读本仓库文件）：把模板顶部"必读上下文"列的那几个 `.md` 文件内容，一起复制粘进去当上下文。
4. **只验证**：模型会给你"接线 + 编译 + 验证"清单，你照着跑一遍确认行为即可。

> 为什么靠谱：本仓库 `docs/skills/` 已是**专为喂大模型写的知识库**（坐标系、跨帧状态、上报、热重载、接线、坑都在里面）。这些模板的核心作用，就是**强制让模型先读对文档、再按项目约定产出、最后给你验证清单**——把"凭记忆瞎发挥"挡在门外。

## 我要做什么 → 用哪个模板

| 你的需求                                             | 用模板                       |
| ------------------------------------------------ | ------------------------- |
| "**检测到 X 就做 Y**"——单路画面规则（存在/计数/停留/越界/位置/姿态/上报…） | **模板 A：新增通道逻辑** ⭐ 最常用     |
| "**多路一起看**"——跨摄像头联动、全局汇总/计数、周期巡检                 | **模板 B：新增全局逻辑**           |
| 改一个**已有逻辑**的行为，或**删掉**一个逻辑                        | **模板 C：修改/删除现有逻辑**        |
| 加可调参数、调阈值、给某逻辑加一个网页能改的数值/开关                       | **模板 D：给逻辑加可调参数**         |
| 网页控制台（前端/后端接口）、部署装包、后台微服务、运维                      | **模板 E：控制台 / 服务 / 运维**    |
| "**改了不生效**" / 报错 / 行为不对，要定位                       | **模板 F：调试 / 定位**          |

> 拿不准：只要是在写"`logic_xxx` 检测规则"就用 A/B/C/D；其它一律先用 E。

## 通用填写技巧（写出好需求 = 拿到好代码）

模型最怕"需求含糊"。动笔前先把这 6 个维度想清楚，缺一个就可能猜错：

| 维度       | 要说清楚的                          | 反面（含糊）→ 正面（清楚）                          |
| -------- | ----------------------------- | ---------------------------------------- |
| **目标类别** | 检测哪个 label（**大小写/拼写须和模型 labels.txt 完全一致**） | "检测人" → "检测 `person`" |
| **区域**   | 全屏，还是只在 ROI（网页画的多边形）内         | "区域内" → "只在 ROI 内（用户在网页画的区域）"            |
| **触发条件** | 存在 / 数量阈值 / 停留时长 / 越界 / 位置关系 / 姿态 | "有人停留" → "在 ROI 内连续停留 ≥ 3 秒"             |
| **动作**   | 画框/文字/染色 + 是否报警 + 是否上报        | "报警" → "命中框染红 + 左上角显示红字 + server 上报带框抓拍" |
| **上报**   | 不上报 / server（HTTP 抓拍）/ dify（AI 核验） | "上报" → "走 server，传带框图"                  |
| **限频**   | 多久最多触发一次（**必须有**，否则一秒几十条）     | "上报" → "同一告警 5 秒最多上报一次"                  |

填模板时把这些直接写进 `【】`。**说不准的地方**（停留多久？发哪种上报？）可以在需求里写"请就此向我确认"，让模型先问再写。

# 模板 A：新增通道逻辑（channel logic）⭐

> 适用："检测到 X 就做 Y"的**单通道、每帧**规则。复制下面整段（含分隔线内全部），填 `【】`，发给模型。

````text

任务：按下面的需求，新增一个【通道逻辑 channel logic】（logic_xxx），并接好线，让我只需验证。

## 动手前必读（严格遵循其中约定，不要凭记忆发挥）
请先阅读以下项目文档，按它们的约定写代码：
- docs/skills/rk3588-channel-logic/SKILL.md          （总览：需求拆解→骨架→接线三件套→验证→坑）
- docs/skills/rk3588-channel-logic/references/channelcontext-api.md   （ctx 字段/查询/绘制/跨帧状态）
- 如需上报：docs/skills/rk3588-channel-logic/references/upload-and-wiring.md
- 如需网页可调参数：docs/skills/rk3588-channel-logic/references/adding-config-parameter.md
- 动手前，从 docs/skills/rk3588-channel-logic/references/examples/ 里挑一个最接近我需求的现有逻辑，照它的范式改。

## 我的需求
一句话需求：【例：ROI 内有 person 连续停留超过 N 秒，就把框染红并上报服务器一张带框抓拍】

- 逻辑名 logic_xxx（英文，小写下划线）：【logic_】
- 网页显示名 label（中文）：【】
- 目标类别（须与模型 labels.txt 完全一致）：【person】
- 检测区域（全屏 / 仅 ROI 内 / 多个命名区域）：【仅 ROI 内】
- 触发条件（存在/计数阈值/停留时长/越界/位置关系/姿态…）：【在 ROI 内连续停留 ≥ N 秒】
- 命中后的动作（画框/文字/染色等可视化）：【命中框染红；左上角显示状态文字】
- 是否上报（不上报 / server / dify）：【server，上报带框抓拍图】
- 限频/冷却规则（必须有）：【同一告警最多每 N 秒上报一次】
- 需要网页可调的参数（名/类型/默认/范围，没有就写"无"）：【dwell_sec int 默认3 范围1-60；report_interval_sec int 默认5 范围1-3600】
- 模型类型（det 检测框 / pose 关键点 / seg 掩码）：【det】
- 其它特殊说明：【无】

## 产出要求（全部对齐项目现有约定）
1. 新建 rk3588_yolo/src/logic/logic_xxx.cpp：
   - 顶部 #include "logic_common.h"；实现 static void logic_xxx(ChannelContext *ctx)；
   - 文件最末尾写一行 REGISTER_LOGIC("logic_xxx", logic_xxx); 自注册（不要改 channel_logic.cpp）；
   - 文件头加 @brief 注释，写清"行为 + 用到的知识点"（参照 src/logic/logic_lessonN.cpp 的风格）。
2. 在 rk3588_yolo/src/logic/logics.json 的 channel_logics 数组加一条声明：
   { "name": "logic_xxx", "label": "中文名", "report": "server 或 dify（不上报就不写此键）", "params": [...] }
3. 若有可调参数：四处 key 完全对齐——
   config.h 的 ChannelConfig 加字段(给默认值) == config_init.cpp 加 REG_C == logics.json 的 param.key == 逻辑里 ctx->config->key。
4. 必须遵守的硬约定：
   - 坐标系一律是「模型输入尺寸」(通常 640×640)：ctx->results[].box、ctx->roi、所有 draw_* 都用它，别拿源视频分辨率算。
   - 跨帧记忆（计时/闩锁/去重）放 ctx->state（每通道独立的 shared_ptr），绝不用 static / 全局变量，以隔离不同通道的变量。
   - 上报地址不硬编码：从 ctx->config->server_url / dify_api_url / dify_api_key 取；上报必须用 ctx->timestamp_ms 限频。
   - 参数每帧从 ctx->config 现读，不要缓存（否则热重载失效）。
   - ctx->frame / ctx->results / ctx->roi 可能为空，先判空。
5. 不要改任何设备的 config.json（那是运行配置，由网页管）。
6. （推荐）照 references/examples/ 的格式补一篇 docs/skills/rk3588-channel-logic/references/examples/logic_xxx.md。

## 最后给我一份收尾说明
1. 这个逻辑怎么在某个通道生效（网页画布该通道的「逻辑函数」节点选 logic_xxx；若上报，连一个「上报配置」节点并填地址）。
2. 编译装包命令：cd rk3588_yolo && ./build.sh <程序名> && sudo ./install_app.sh <程序名>，然后网页「程序管理」重启。
3. 怎么验证行为：看监看画面/overlay；server 上报看 <App>/alarm_store/ 是否落盘，dify 看 redis-cli lrange dify_queue 0 -1。
4. 若用到 ROI：改了 ROI 必须在「程序管理」停止再启动（ROI 只在启动时加载，不热重载）。

如果上面需求里有歧义（如停留多久、发哪种上报），先简短问我确认，再开始写。
````

# 模板 D：给某逻辑加一个网页可调参数

> 适用：给已有逻辑（或正在写的逻辑）加一个用户能在网页改的数值/开关（半径、停留秒数、阈值、开关…）。

````text
你是本项目 rk3588_visual_analysis_framework 的 C++ 二次开发工程师。
任务：给逻辑【logic_xxx】加一个网页可调参数，并保证热重载生效。

- docs/skills/rk3588-channel-logic/references/adding-config-parameter.md  （四处对齐 + 热重载机制 + 坑）

## 我的需求
- 参数加到哪个逻辑：【logic_xxx】
- 参数键名 key（英文小写下划线）：【dwell_sec】
- 类型：【int / float / bool / string / enum】
- 默认值：【3】
- 范围 min/max（数字才有）或 enum 选项：【1 ~ 60】
- 网页显示名 label（中文）+ 帮助说明 help：【停留秒数 / 目标在 ROI 内连续停留超过此秒数即报警】

## 最后给我收尾说明
````

# 模板 F：调试 / 定位（"改了不生效" / 报错 / 行为不对）

> 适用：东西没按预期工作，要系统定位。

````text
你是本项目 rk3588_visual_analysis_framework 的工程师，帮我定位一个问题。

## 动手前必读

- 若问题在某个检测逻辑里：docs/skills/rk3588-channel-logic/SKILL.md 的「常见坑」一节。

## 现象
- 我做了什么改动：【例：改了 logic_xxx 的阈值 / 加了个参数 / 换了模型】
- 期望行为：【】
````
